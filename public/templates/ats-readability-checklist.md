# ATS-Readability Checklist
### A free pre-flight from ResumeValue

> Goal: make sure software reads your resume CLEANLY. (No tool can promise you'll "pass" a specific employer — that's keywords + their settings.)

## Format
- [ ] Single column (no tables, text boxes, or multi-column layouts)
- [ ] Contact info in the body, NOT in the header/footer
- [ ] Standard fonts; real selectable text (not an image/scan)
- [ ] No skill graphics, rating bars, or icons carrying important info

## Structure
- [ ] Conventional headings: Summary · Experience · Skills · Education
- [ ] Each role has title, employer, and dates in a consistent format
- [ ] Skills written out as words (not only inside a graphic)

## File
- [ ] Saved as a text-based PDF or DOCX (open it, try to select the text)
- [ ] File name is plain (yourname-resume.pdf)

## Keywords (honest)
- [ ] Real job-description terms used ONLY where genuinely true of me
- [ ] No keyword-stuffing for skills I don't have

## Final check
- [ ] I could paste this resume into plain text and it still makes sense in order
